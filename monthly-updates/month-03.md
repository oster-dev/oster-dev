**Projects That Prove Month 3**

**Retail Data PySpark ETL**

This repository is the main proof point for Month 3. It is a small but complete PySpark ETL project focused on reading retail transaction data, applying core cleaning and transformation steps, and writing analytical output in a structured way.

The project was intentionally kept small. The goal was not to build a production-grade platform, but to create a focused Spark project that proves practical understanding of Spark DataFrames, transformations, timestamp handling, aggregations, and basic ETL structure.

That makes it a direct match for the roadmap requirement: a first Spark ETL project in Python, publicly visible on GitHub.

**What Was Covered**

**AWS DEA-C01**  
Month 3 introduced a much more data-engineering-focused AWS learning phase. Compared with CLF-C02, this required deeper exposure to storage, ingestion, transformation, and operational concepts relevant to real data systems.

**Spark**  
The main Spark concepts covered were DataFrames, RDDs, transformations, and actions. The focus was on understanding how Spark processes distributed data and how ETL logic is structured in a reproducible pipeline.

**Scala Reading**  
The goal was not writing Scala from scratch. The goal was reading and understanding Spark-Scala code well enough to follow structure, logic, and adaptation patterns. That matters because Scala-heavy Spark environments are common in larger data platforms.
